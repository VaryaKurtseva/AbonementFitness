# AbonementFitness
Проект на джаве по предмету Системно-орентированное программирование
Предметная область: Система управления абонементами в фитнес-клуб 
Типовой сценарий: Пользователь хочет продлить свой абонемент, но не может из-за нарушений правил клуба. 
1. Пользователь нажимает на кнопку «Продлить абонемент». 
2. Основной сервис сообщил об этом событии в шину. 
3. Сервис аудита записал данное событие в лог. 
4. gRPC клиент спросил у gRPC сервер репутацию пользователя, может быть он нарушил правила фитнес клуба, может быть он вел себя неподобающее во время занятий спортом. 
5. Основной сервис увидел это сообщение и перевел статус запроса на «отказано», отправив об этом сообщение в шину.
6. Сервис уведомлений отправил уведомление клиентскому приложению об отказе в бронировании.
